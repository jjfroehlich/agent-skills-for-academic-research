# Introductions

Use this when the user needs help with an introduction, gap statement, significance paragraph, aims paragraph, scope statement, or thesis opening.

## The job of this topic

An introduction moves readers from shared territory to a specific unresolved problem, then shows why the present work is the necessary next step.

## Default workflow

1. Name the audience and how much background they already have.
2. Establish the territory with only the context needed to understand the problem.
3. Narrow to the specific gap, uncertainty, limitation, contradiction, or unresolved tension.
4. Explain why the gap matters: what field, community, practice, or theory is limited until it is addressed.
5. State the aim, objective, question, hypothesis, or contribution, keeping those categories distinct.
6. Preview the approach, scope, limitations, or structure only as much as needed for orientation.
7. For thesis introductions, add only the required broader components: research problem, questions, significance, methodology overview, limitations, and chapter map.

## Decision rules

- If the opening is a broad field survey, compress it to the facts needed for the gap.
- If the gap is phrased as "little is known," specify what is unknown, for whom it matters, and why prior work is insufficient.
- If the aim appears before the reader understands the problem, move or add context.
- If the introduction reads like a thesis literature review, separate essential motivation from broader background.
- If the final paragraph overpromises, align the stated contribution with the actual results or chapter scope.
- If a significance paragraph lists beneficiaries generically, name the benefit mechanism and move from broad contribution to specific beneficiary.
- If aims, objectives, and questions blur together, rewrite the aim as purpose, objectives as tasks, and questions as interrogatives.
- If a thesis introduction was written before the project matured, revise it after the chapters are known so the roadmap matches the actual contribution.
- If the opening contains a contradiction or field tension, make that tension the narrowing path instead of adding more background.

## Variants and edge cases

- Thesis introductions can tolerate more background than paper introductions, but still need a visible narrowing path.
- Significance sections should distinguish scientific significance from personal or institutional motivation.
- PhD introductions may need background, research problem, questions, significance, methodology overview, limitations, and a thesis outline; write or revise them after the thesis contents are known.
- Review introductions should name the organizing problem and selection logic.
- Multidisciplinary introductions should define only terms needed for the argument.

## Anti-patterns

- Starting with a dictionary-style definition.
- Citing many facts without explaining the tension between them.
- Claiming novelty without naming the prior-work boundary.
- Previewing every section in a way that reads like a table of contents.
- Using hype to compensate for a weak gap.

## Diagnostic questions

- What does the reader need to know before the gap makes sense?
- Which sentence states the gap?
- Which sentence states why the gap matters?
- Which sentence states what this work does about it?
- Are aim, objectives, questions, and hypothesis doing distinct jobs?
- Does the roadmap reflect the actual artifact rather than a generic table of contents?
- What background can be moved to literature review, methods, or supplement?

## Output patterns / mini-templates

- Introduction arc: `Known context. Persistent uncertainty or limitation. Why that limitation matters. Here we address it by...`
- Gap repair: `Prior work established [X], but [specific unresolved problem] remains unclear because [reason].`
- Aim sentence: `Here, we [approach] to test/measure/resolve [specific question] in [system/context].`
- Significance sentence: `By resolving [gap], this study provides [new knowledge/use] for [field/beneficiary] because [mechanism].`
- Thesis opening map: `Background -> research problem -> significance -> question/aim -> approach -> scope/limits -> chapter roadmap.`

## Examples

Before: `Many pathways regulate cell identity, and this has been studied for decades.`

After: `Although several pathways have been linked to cell identity, it remains unclear which regulatory changes occur before the first stable lineage decision.`

## When not to apply this

Do not use introduction guidance to rewrite a discussion as if it were a setup section; discussions should interpret evidence already presented.
