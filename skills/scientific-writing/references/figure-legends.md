# Figure Legends

Use this when the user asks for figure captions, panel descriptions, figure-title logic, statistical annotation wording, or self-contained legend review.

## The job of this topic

A figure legend lets readers understand what the figure shows, how to map panels and symbols to evidence, and what comparison matters without relying on the main text.

## Default workflow

1. Identify the figure's main takeaway or purpose and whether this is an initial submission legend or a final journal-condensed legend.
2. Draft a concise title or opening phrase that names the finding when supported.
3. Describe panels in order and define labels, colors, symbols, abbreviations, and conditions.
4. Include essential methods, sample size, replicate information, and statistical notation when supplied.
5. For initial submission, allow a brief takeaway or local interpretation that helps reviewers orient to the figure.
6. For final publication, condense toward journal style and move detailed workflow explanation to main text or Methods when required.
7. Check that every panel reference has a visible referent and every main-text claim points to the right panel.
8. Run a standalone-inspection pass: a reviewer should be able to inspect the figure without guessing the comparison, condition, symbol, or statistical convention.

## Decision rules

- If a legend only repeats axis labels, add the biological or analytical comparison.
- If the main text depends on the figure, ensure the legend supplies enough context to inspect the evidence.
- If statistical symbols are present, define the test or threshold only when supplied by the draft or figure.
- If a panel is not discussed in the text, decide whether it is needed or whether the text needs a callout.
- If the legend becomes a mini-results section, cut broad interpretation and keep orientation.
- If the legend says only "Schematic of..." for an initial submission, add the workflow purpose or takeaway; for final publication, follow journal condensation rules.
- If the user supplies no sample size, replicate, or statistical-test information, flag the missing fields rather than inventing them.
- If panel phrasing is inconsistent, use parallel structure for panels that make parallel comparisons.

## Variants and edge cases

- Multi-panel figures need a consistent panel order and parallel phrasing.
- Supplementary legends can include more procedural detail than main figures.
- Initial-submission legends can be more reader-helpful than final legends because editors and reviewers often inspect figures first.
- Schematics need definitions of visual conventions even when no statistics appear.
- Methods figures may need workflow steps rather than result claims.

## Anti-patterns

- Undefined colors, symbols, abbreviations, or error bars.
- Panel letters that do not match the visual sequence.
- Claiming significance without visible statistical support.
- Long legends that repeat the entire results paragraph.
- Figure titles that name only the method when the result is clear.

## Diagnostic questions

- What should the reader learn from the figure?
- Can each panel be understood independently?
- Are conditions, controls, sample size, and statistics clear enough for inspection?
- Does the legend define every non-obvious visual element?
- Do main-text citations and legend panel order match the visual order?
- Does the legend support, but not overstate, the main-text claim?

## Output patterns / mini-templates

- Legend structure: `Takeaway/title. (A) What panel A shows and key condition. (B) What panel B compares. Data/statistics note if supplied. Abbreviations and symbols.`
- Revision note: `Define [symbol/color], add [condition/comparison], cut [duplicated interpretation], and align [panel] with main-text claim.`
- Missing-info note: `Needed before final legend: [sample size/replicates/statistical test/error bar definition/abbreviation definition].`

## Examples

Before: `Figure 2. Results of assay.`

After: `Figure 2. Treatment increases marker expression in late-stage cells. (A) Representative images of control and treated cells. (B) Quantification of marker-positive cells across the indicated conditions.`

## When not to apply this

Do not invent methods, sample sizes, or statistical tests from a figure legend request. Ask for the missing figure details or mark assumptions.
