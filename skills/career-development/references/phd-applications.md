# PhD Applications

Use this when the user is applying to PhD programs, choosing between offers, preparing visit questions, or deciding whether a graduate-training environment fits.

## The job of this topic

Help the user evaluate graduate training as a long, high-dependency environment. A good answer compares programs by mentorship, funding, project flexibility, milestones, community, and exit options, not only by ranking or research topic.

## Default workflow

1. Identify the user's field, stage, target programs, constraints, and whether the task is application strategy or offer comparison.
2. For applications, align the statement and CV with research fit, preparation, questions the user can pursue, and evidence of readiness.
3. For program choice, compare advisor access, rotations, funding guarantees, teaching load, cohort support, qualifying exams, publication expectations, and career outcomes.
4. Prepare questions for prospective advisors, current students, program administrators, and alumni.
5. Flag structural risks: unclear funding, no fallback advisor, weak student community, excessive service load, or milestones that do not match the user's needs.
6. If the user is considering leaving or changing a PhD path, separate program-structure evidence from acute distress, durable mismatch, and practical constraints.
7. End with a ranked decision matrix, a list of missing evidence, or a support-and-information plan before any irreversible step.

## Decision rules

- If a program depends on one advisor only, check fallback advisors and department support before accepting.
- If funding is unclear after year one, get specifics on stipend duration, grant dependency, teaching requirements, and emergency support.
- If current students independently name the same concern, treat it as evidence, not gossip.
- If the user's interests are still broad, value rotations, methodological breadth, and a program structure that preserves options.
- If the user has hard geographic, health, family, financial, or visa constraints, include those constraints in the decision rather than treating them as secondary.
- If the user is considering leaving, list reversible and irreversible actions, map financial and administrative constraints, and identify neutral support before recommending a move.

## Variants and Edge Cases

- For statement reviews, focus on fit, evidence of preparation, research questions, and advisor/program specificity.
- For visit days, design different question sets for faculty, current students, administrators, and alumni.
- For interdisciplinary students, check whether co-advising, coursework, and departmental expectations are compatible.
- For students considering leaving a PhD, use this file for graduate-program structure, advisor options, milestones, funding, and support planning; use `career-transitions.md` when the main issue is comparing non-PhD paths.

## Anti-Patterns

- Ranking programs only by prestige.
- Asking only "What is the culture like?" instead of asking for specific practices and recent examples.
- Ignoring funding, visa, family, health, or location constraints until after acceptance.
- Writing an application that lists interests without showing preparation or fit.
- Assuming a good research topic compensates for poor advising structure.
- Treating a leaving-PhD question as simple persistence advice instead of a high-dependency decision with missing evidence and support needs.

## Diagnostic Questions

- What research questions does the user want to be able to pursue after year one?
- Which advisors could plausibly supervise the work, and what happens if the first match fails?
- How are rotations, qualifying exams, teaching, funding, and dissertation milestones structured?
- What do current students say about feedback, workload, support, and conflict?
- What outcomes do recent alumni have, and are those outcomes compatible with the user's goals?
- Is the current problem acute distress, durable mismatch, missing structure, advisor conflict, or an external constraint?

## Output Patterns / Mini-Templates

Program comparison row:

| Program | Advisor fit | Funding clarity | Flexibility | Student reports | Main risk | Next question |
|---|---|---|---|---|---|---|
| Program A | Strong with two possible labs | Five years stated | Rotations available | Good feedback, heavy teaching | Time pressure | Ask how teaching affects year two |

Visit-day question:

> Can you give me an example of a student who changed projects or advisors, and what support the program provided?

Leave/stay decision memo:

1. Current problem: acute crisis, durable mismatch, advisor/lab conflict, funding constraint, or unclear evidence.
2. Reversible options: change project, add committee support, take leave, transfer labs, reduce load, or run a parallel search.
3. Irreversible or hard-to-reverse options: master out, resign, move, change visa status, or abandon a funded slot.
4. Support plan: neutral mentor, program director, ombuds, clinician, trusted peer, or financial/visa adviser as appropriate.
5. Lowest-regret next step for the next one to two weeks.

## Examples

- Good application evidence: a prior project, method, or question that shows the user can enter the proposed research area.
- Good offer-comparison evidence: funding terms in writing plus student reports about how often those terms hold in practice.
- Weak evidence: a program website says "collaborative" but no student can name how collaboration is supported.
- Good leaving-path framing: "I need to verify whether this is an advisor-specific problem, a program-structure problem, or a career-goal mismatch before I decide."

## When not to apply this

Do not use this playbook for faculty hiring, postdoc searches, or generic college admissions. Use it for doctoral training decisions and materials.
