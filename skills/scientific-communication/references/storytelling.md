# Storytelling

Use this when the user needs better narrative flow, audience memory, explanation, framing, or a stronger through-line for scientific content.

## The job of this topic

Turn facts into a clear explanatory journey without oversimplifying the science. The story should create orientation, tension, resolution, and meaning while preserving uncertainty.

## Default workflow

1. Identify the audience's starting belief or confusion.
2. Name the central question, gap, conflict, or decision.
3. Arrange content as `context -> problem -> attempt -> evidence -> interpretation -> implication`.
4. Use examples, analogies, or visuals only when they clarify a relationship.
5. Make transitions explicit so the audience sees why each step follows.
6. Keep uncertainty and limitations near the claims they qualify.
7. End with what the audience can now understand, decide, or ask.

## Decision rules

- If the draft is a list of results, turn each result into an answer to a question.
- If the story is too broad, choose one audience-relevant tension.
- If an analogy is helpful, state where it maps and where it stops.
- If the user wants excitement, add stakes and contrast without inflating claims.
- If the conclusion feels weak, connect it back to the opening problem.

## Variants and edge cases

- Discovery story: emphasize the gap and sequence of evidence.
- Mechanism explanation: build the causal chain step by step.
- Translational story: connect mechanism to decision, patient, product, or policy implications.
- Negative or null result: frame the story around what uncertainty was reduced.
- Public explanation: use familiar stakes and concrete examples before abstraction.

## Anti-patterns

- Adding drama that the evidence does not support.
- Hiding the main point until the end when the audience needs orientation.
- Using a cute analogy that creates false scientific inferences.
- Explaining every exception before the main model is understood.
- Treating storytelling as decoration rather than structure.

## Diagnostic questions

- What question keeps the audience listening?
- What did the audience believe before, and what should change?
- Which evidence is decisive for the story?
- Where does uncertainty belong?
- What is the simplest valid version of the explanation?

## Output patterns / mini-templates

Story spine:

```text
Before:
Problem/gap:
Attempt or method:
Key evidence:
What changed:
So what:
```

Explanation audit:

```text
Audience starting point:
Terms to define:
Causal steps:
Example/analogy:
Known limits:
Usable takeaway:
```

## Examples

- Turn "we measured X, Y, and Z" into "we needed to know whether mechanism A or B explains the effect; X ruled out A, Y supported B, and Z shows the boundary."
- For a public explanation, begin with what the phenomenon changes for people, then introduce the scientific mechanism.
- For a null result, make the story about the decision the result now prevents or the hypothesis it narrows.

## When not to apply this

Do not use narrative framing to hide weak evidence, remove necessary caveats, or replace technical precision where precision is required.
