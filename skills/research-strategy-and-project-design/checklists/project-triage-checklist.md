# Project Triage Checklist

Use this checklist for a fast project decision. Keep it practical: the goal is to expose the next decision, not to make a comprehensive plan.

## Candidate Project

- What is the concrete question or output?
- What would be learned if it worked?
- What makes it feasible with current tools, data, people, and time?
- What makes it risky?
- What alternative project competes for the same effort?

## Portfolio And Stage Fit

- Does the project provide traction, training, independence, upside, or infrastructure?
- Is its risk appropriate for the user's career stage and decision horizon?
- Does the portfolio already contain too many similar risks?
- Can a grand challenge be split into a smaller decisive project?

## Decisive Uncertainty

- This project is worth continuing only if: `[condition]`.
- The most likely failure mode is: `[failure mode]`.
- The cheapest credible check is: `[test/search/pilot/control]`.
- Continue if: `[threshold]`.
- Stop or pivot if: `[threshold]`.

## Evidence-Type Audit

For each major claim, record:

| Claim | Current evidence | Scale covered | Blind spot | Complementary check |
|---|---|---|---|---|
| | | | | |

Add complementary evidence only if it changes the decision or interpretation.

## Branch Review

- What new branches have opened?
- Which branch has the highest expected learning per cost?
- Which branch should be parked explicitly?
- Who or what capability would make a parked branch worth reopening?

## Recommendation

End with one of:

- Pursue now.
- Run a cheap decisive test first.
- Narrow the question.
- Pivot to a stronger branch.
- Pause until a missing dependency is solved.
- Stop and redirect time.
