# Reviewer Ethics

Use this when the user asks about conflicts of interest, confidentiality, use of unpublished information, suspected misconduct, confidential comments, review transfer, open peer review, or whether to accept a review.

## The job of this topic

Protect the integrity of the review process while still helping the reviewer produce useful feedback. Ethics guidance is a boundary layer around peer-review content.

## Default workflow

1. Identify the ethical issue: conflict, competence, confidentiality, privileged information, misconduct concern, review reuse, open review, or confidential communication.
2. Check journal instructions and reviewer invitation terms first.
3. Decide whether the reviewer can accept, must decline, should disclose, or should ask the editor.
4. Keep unpublished manuscript information confidential unless policy and consent clearly allow sharing.
5. Before involving a student, colleague, co-reviewer, or review-transfer workflow, get editor or journal permission and name contributors when required.
6. Put sensitive concerns in the editor-only channel when appropriate.
7. Avoid public accusations, personal use of privileged information, citation coercion, or unapproved sharing with colleagues or trainees.

## Decision rules

- If a real or perceived conflict could affect judgment, disclose it before accepting or decline.
- If you need help from another person to review, ask the editor before sharing the manuscript.
- If you suspect misconduct, plagiarism, image manipulation, or serious ethics problems, tell the editor with evidence and avoid author-facing accusations.
- If a review may be signed, published, or transferred, follow the journal's consent and confidentiality rules and write with public-context clarity.
- If the manuscript overlaps with your own unpublished work, disclose the overlap and avoid using privileged information.
- If you cannot complete the review on time, tell the editor early.
- If recommending citations, make sure they are necessary, relevant, and proportionate rather than serving reviewer self-interest.

## Variants and edge cases

- **Trainee involvement:** Involving trainees can be educational only when journal policy permits it and confidentiality is preserved.
- **Open review:** Public readability increases the need for context, fairness, and avoidance of language that can be weaponized.
- **Review transfer:** Reassess comments against the new journal's standards and remove comments that only applied to the previous venue.
- **Competing work:** A reviewer can be expert and conflicted at the same time; disclose and let the editor decide.
- **Confidential comments:** Use them for decision-relevant synthesis or sensitive process issues, not for hidden criticism authors should address.

## Anti-patterns

- Sharing manuscripts with lab members without editor approval.
- Using ideas, data, or methods from a manuscript before publication.
- Accepting a review to delay or influence a competitor.
- Hiding conflicts because they seem obvious or minor.
- Making unsupported accusations in author-facing comments.
- Publishing or reusing review content without consent.
- Recommending unnecessary citations to benefit the reviewer or associates.

## Diagnostic questions

- Would the editor want to know this relationship, competition, or prior involvement?
- Can I review the manuscript without using privileged information?
- Does journal policy allow sharing the manuscript with anyone else?
- Is this concern something authors can fix, or something the editor must investigate?
- Would public release of this review expose confidential or unfairly framed information?
- Am I separating ethical concerns from scientific disagreement?

## Output patterns / mini-templates

**Conflict disclosure**

```text
Before accepting, I should disclose that [relationship/competition/prior involvement]. I can provide an objective review on [areas], but I understand if you prefer another reviewer.
```

**Editor-only ethics note**

```text
Confidential note to editor: I noticed [specific concern] in [location]. I cannot determine intent from the manuscript, but it may warrant editorial review before a decision.
```

**Decline note**

```text
Thank you for the invitation. I should decline because [conflict/expertise/deadline]. If helpful, possible reviewers include [names or expertise areas].
```

## Examples

- If a reviewer recognizes a collaborator's work in a double-blind process, the next step is disclosure to the editor, not silent continuation.
- If a lab member has the right expertise, the reviewer still needs journal permission before sharing the manuscript.

## When not to apply this

Do not use this playbook to make legal or institutional misconduct determinations. It helps the agent route concerns to the editor and follow journal policy.
