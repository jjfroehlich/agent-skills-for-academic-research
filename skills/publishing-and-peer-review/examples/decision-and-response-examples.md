# Decision And Response Examples

Use these patterns for publication-process advice.

## Decision letter triage

Input: The editor rejects the manuscript but offers transfer with reviews.

Output pattern:

```text
Decision type: rejection with a transfer path, not a revise-and-resubmit.
First question: do the reviews identify fixable issues that would also matter at the transfer journal?
Options: transfer if the target journal fits and the required revision is bounded; submit elsewhere if the transfer narrows audience or carries unfixable framing problems.
Next action: make a response-independent revision plan before writing any cover letter.
```

## Reviewer response triage

Input: Reviewer 2 asks for one feasible control, one expensive new experiment, and one wording change.

Output pattern:

```text
Accept: perform or add the feasible control if it directly tests the critique.
Negotiate: explain why the expensive experiment is outside scope and add a limitation if appropriate.
Edit: make the wording change directly and cite the revised location.
Tone: thank the reviewer for the concern, then answer with evidence rather than defensiveness.
```
