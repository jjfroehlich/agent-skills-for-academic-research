# Funder Fit And Compliance

## Use this when...

Use this when the user is choosing a funding opportunity, checking call fit, interpreting instructions, preparing submission materials, or auditing compliance.

## The job of this topic

Prevent unreviewable or low-fit applications before narrative polish. Treat the call, templates, assessor guidance, budget rules, and internal deadlines as part of the scoring system.

## Default workflow

1. Identify the funder, mechanism, call purpose, eligibility, deadline, and review criteria.
2. Check whether the project type, applicant stage, location, budget size, and outputs fit the call.
3. Convert instructions, templates, formatting, section limits, portal requirements, and assessor criteria into a checklist.
4. Map each assessor criterion to a section, label, or paragraph in the proposal.
5. Use research-office or experienced-colleague support for local rules, examples, budgets, graphics, and deadlines.
6. Repeat the preflight before submission because familiar calls can change.

## Decision rules

- If the project does not match the call scope, stop or retarget before revising prose.
- If eligibility or format is uncertain, resolve it before drafting final text.
- If the template is supplied, follow it exactly.
- If a review-sheet answer is hard to find, add a heading, label, or explicit sentence.

## Variants and edge cases

- Some mechanisms use broad panels, some use close-field reviewers, and some have interview stages; tune the proposal to the actual process.
- Prior successful applications can help reveal expected structure, but current instructions override examples.
- Internal deadlines and institutional approvals can be binding even when funder deadlines are later.

## Anti-patterns

- Drafting a full proposal before checking scope and eligibility.
- Assuming last year's rules still apply.
- Altering templates or formatting to gain space.
- Treating CV, impact, DEI, budget, or institutional forms as administrative afterthoughts.

## Diagnostic questions

- Does this call fund this kind of work?
- Is the applicant eligible this year?
- Which template, formatting, or portal rule could make the proposal noncompliant?
- Where does the proposal answer each assessor criterion?
- Which internal approval or budget rule could block submission?

## Output patterns / mini-templates

Preflight table:

```text
Call scope:
Eligibility:
Required sections:
Templates/format:
Assessor criteria:

```

## Examples

Weak preflight: "The science is strong, so this funder is worth trying."

Stronger preflight: "The call funds early-stage method development, the applicant is eligible, the budget cap supports the work packages, and each assessor criterion has a mapped section."

## When not to apply this

Do not infer official rules from general grant-writing advice. Ask the user for the current call text or state which compliance items remain unresolved.
