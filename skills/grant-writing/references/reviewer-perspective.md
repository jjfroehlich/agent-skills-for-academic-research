# Reviewer Perspective

## Use this when...

Use this when panel readability, scoring criteria, non-specialist reviewers, external assessors, skim value, reader feedback, or interview preparation are central.

## The job of this topic

Reduce reviewer labor. Make the proposal easy to score, summarize, defend in panel, and trust under time pressure.

## Default workflow

1. Identify the likely reviewer breadth: close specialists, broad panel, interdisciplinary assessors, or interview panel.
2. Translate review criteria into findable claims and labels.
3. Add context and definitions where non-specialists may otherwise guess.
4. Run a skim test on headings, topic sentences, figures, and intentional bold statements.
5. Ask what objection a skeptical reviewer would raise, then add evidence or restructure the draft.
6. Get critical readers early enough to revise, not just proofread.

## Decision rules

- If a reviewer cannot recover the logic from headings, topic sentences, and figures, rewrite those signposts before polishing sentences.
- If a required review criterion is hard to find, add a label or move the answer.
- If broad reviewers may not know the field, add enough context and key literature to orient them without overloading the page.
- If the mechanism has an interview, prepare short answers and backup evidence for likely doubts instead of repeating the proposal.

## Variants and edge cases

- For broad panels, define terms and make significance concrete before technical depth.
- For interview mechanisms, map each likely doubt to a concise answer and a backup slide or evidence point.
- For final revision, use bold or emphasis only when it helps reviewers skim the scoring argument.

## Anti-patterns

- Writing only for insiders when the proposal is panel-triaged.
- Random bolding that does not form a coherent skeleton when read alone.
- Decorative figures that do not carry claims.
- Requesting feedback too late for structural revision.
- Assuming reviewers will infer compliance from scattered prose.

## Diagnostic questions

- Can a busy reviewer state the objective after the first page?
- Can a non-specialist score significance, innovation, approach, and feasibility?
- Can the proposal be summarized from headings, topic sentences, and figures?
- Which scoring question is hardest to answer?
- Where did external readers become lost, skeptical, or unconvinced?

## Examples

Weak signpost: "Approach 1: Sequencing diverse samples."

Stronger signpost: "Aim 1 establishes whether [signal] is conserved across [sample classes], creating the basis for the intervention test in Aim 2."

## When not to apply this

Do not replace official review criteria with generic reviewer psychology. Use this playbook to make the current criteria easy to score.
