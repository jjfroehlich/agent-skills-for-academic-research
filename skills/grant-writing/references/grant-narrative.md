# Grant Narrative

## Use this when...

Use this when the proposal's central story is unclear, the objective appears too late, sections feel disconnected, or the user needs a reviewer-facing outline.

## The job of this topic

Make the proposal easy to understand as one funding argument: a problem worth solving, a clear objective, an active premise, a plausible approach, and outcomes reviewers can score.

## Default workflow

1. Write the objective in one sentence: "We will [test/build/define] [object] to [advance/solve] [need]."
2. Write the active premise: the central claim, mechanism, resource, or intervention the proposal will test or make possible.
3. Put the objective and premise before extended background.
4. Build the section order around reviewer logic: problem, significance, gap or enabling advance, approach rationale, proposed work, interpretable outcomes, risks, and long-term direction.
5. Cut background that proves reading breadth but does not help a reviewer score importance, novelty, feasibility, or fit.

## Decision rules

- If the objective appears only after background, move it to the opening and make it concise.
- If the proposal reads as a list of activities, rewrite around one active premise and the decisions each aim will enable.
- If the draft uses a novel structure, check whether the call expects a recognizable scoring structure before preserving it.
- If logic is hard to follow, fix the argument before adding technical detail.

## Variants and edge cases

- Fellowships need explicit "why this applicant," "why this lab/environment," and "why now" logic.
- Early-career project grants may need the CV or track record to support future execution, not dominate the narrative.
- Mechanisms with strict templates should keep the narrative spine while using the funder-required headings.

## Anti-patterns

- Starting with a literature condensation before explaining the problem and premise.
- Hiding the main objective in the middle of the page.
- Using background as a substitute for a proposed action.
- Making the proposal sound like a retrospective achievement report when the mechanism funds future work.

## Diagnostic questions

- What problem should the reviewer care about after the first page?
- What is the one-sentence objective?
- What premise connects the aims to the payoff?
- Which sections answer why this problem, why now, why this approach, and why this applicant or environment?
- Which paragraph can be cut without changing the scoring argument?

## Output patterns / mini-templates

Active premise map:

```text
Objective:
Active premise:
Why this problem:
Why now:
Why this approach/model/system:
Why this applicant/team/environment:
Expected change if successful:
```

Story order:

```text
Problem -> significance -> gap/enabling advance -> approach rationale -> aims/work packages -> interpretable outcomes -> risks and alternatives -> long-term direction
```

## Examples

Weak: "This proposal studies immune signaling in inflammatory disease."

Stronger: "This proposal tests whether [mechanism] explains [disease-relevant observation] and whether targeting [node] can create a tractable intervention path."

## When not to apply this

Do not override required funder headings. Use the narrative spine inside the required structure.
