# Significance, Innovation, And Approach

## Use this when...

Use this when the user needs help distinguishing significance, innovation, approach, impact, ambition, or high-risk/high-return framing.

## The job of this topic

Separate the scoring arguments: why the work matters, what is new, why the approach is suited to the problem, and why the project is ambitious without being implausible.

## Default workflow

1. State what has been done and what remains needed.
2. Explain why solving the problem matters now.
3. Name what is new: technology, application, model, dataset, mechanism, combination, or conceptual frame.
4. Explain why the proposed approach is better suited than alternatives.
5. For ambitious or high-risk work, distinguish scientific risk from unmanaged execution risk.
6. If impact, engagement, or DEI sections are required, make them specific to the funder's criteria and the project context.

## Decision rules

- If significance only says the topic is important, add the field state, unmet need, and expected change.
- If innovation only says "novel," specify what is new and why that novelty enables the objective.
- If approach is a methods list, add rationale, feasibility evidence, interpretable outcomes, and alternatives.
- If the project is high-risk, show why the return is worth it and what execution risks are already controlled.

## Variants and edge cases

- Do not force "paradigm shift" language if the project does not support it.
- Some funders score broader impacts, engagement, or DEI heavily; others do not ask for them.
- Empirical projects may need concrete data access, representativeness, mergeability, and identification strategy as feasibility evidence.

## Anti-patterns

- Combining significance and innovation into one vague excitement paragraph.
- Claiming impact without a path to users, communities, policies, patients, or scientific practice.
- Treating missing data access or undeveloped analysis strategy as acceptable scientific risk.
- Using boilerplate DEI or engagement text disconnected from the proposal.

## Diagnostic questions

- What has been done, and what is still needed?
- Why does this problem matter now?
- What exactly is new?
- Why is this approach or model system appropriate?
- What is the high return, and which risks are scientific rather than execution gaps?
- Which funder-specific impact or DEI claims are actually supported?

## Output patterns / mini-templates

Significance/innovation split:

```text
Significance: field state
Significance: unmet need
Significance: why it matters now
Innovation: new technology/resource/model/application
Innovation: why that novelty enables the project
Approach: why this path is suited to the problem
```

Risk-return-feasibility row:

```text
Ambitious claim:
Expected return:
Scientific risk:
Execution risk already controlled:
Feasibility evidence:
Reviewer doubt reduced:
```

## Examples

Weak: "This project is innovative because it uses a new platform."

Stronger: "The platform is innovative here because it measures [previously inaccessible variable], allowing the proposal to test [specific mechanism] rather than infer it indirectly."

## When not to apply this

Do not add unsupported impact, DEI, or high-risk claims. If evidence is missing, tell the user what evidence or scope change is needed.
