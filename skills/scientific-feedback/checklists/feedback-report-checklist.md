# Feedback Report Checklist

Use this before returning an integrated scientific feedback report.

## Scope

- Artifact, destination, audience, relationship, and review mode are explicit.
- Selected lenses are named and limited to what the artifact needs.
- Assumptions and missing context are separated from confirmed findings.

## Content

- Strengths worth preserving are named.
- Blocking issues are separated from major revisions, secondary improvements, and optional polish.
- Each concern includes an action the user can take.
- Domain lenses are integrated into one revision sequence rather than listed as separate unrelated reports.
- The report avoids source provenance, bibliography-like material, and fake source-backed rules.

## Failure signals

- The top recommendation is a style change while the artifact has an evidence, compliance, feasibility, audience, or structure problem.
- Feedback comments are generic enough to apply to any artifact.
- The user cannot tell what to do first.
- The response claims to have invoked a skill rather than applying a lens or consulting a skill if available.

## Final pass

```text
Scope:
Selected lenses:
Preserve:
Must fix first:
Major revisions:
Secondary improvements:
Missing context:
Next action:
```
