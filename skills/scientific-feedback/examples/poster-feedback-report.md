# Poster Feedback Report

Use this as an output pattern for poster feedback.

## Pattern

```text
Scope: poster for <conference/audience>.
Selected lenses: scientific-communication; add data-visualization-and-figures for figure readability.

Main claim:
- <what should be visible from a few steps away>

Must fix before presentation:
1. <layout/figure/text issue -> action -> rationale>
2. <accessibility/readability issue -> action -> rationale>

Cut or compress:
- <text/detail to remove or move>

Walkthrough:
- <30-second route through the poster>

Next production check:
- <print/export/readability action>
```

## Weak vs strong

Weak: "The poster has too much text."

Strong: "The methods column is taking the visual priority that should belong to the main result. Cut methods to three bullets, enlarge the result figure, and make the title state the finding."

## Bad assistant behavior

- Reviewing the poster as if readers will read every paragraph.
- Recommending decorative color before fixing hierarchy.
- Ignoring standing-distance readability.
